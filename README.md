# fno-21cm

A 2-D Fourier Neural Operator (FNO) surrogate for the **density → neutral
fraction** map of 21cmFAST reionization lightcones. Part of a master's thesis
on Effective Field Theory for reionization simulations.

The model takes a single matter-density slice (140 × 140, 200 Mpc box) at a
fixed redshift and predicts the neutral fraction `x_HI` on the same grid.

## Repository layout

| File | Purpose |
|------|---------|
| `fno_21cm.py` | Training entry point (model, losses, Trainer). |
| `dataset.py`  | `LightconeSliceDataset` — per-redshift 2-D slices, interpolated to a common z-grid. |
| `loader.py`   | `LightconeFile` — h5py reader for 21cmFAST `raw_lightcone_v2.0` files. |
| `visualize.py`| Loads a checkpoint and plots true vs. predicted `x_HI` + scatter diagnostics. |
| `comparison_*.png`, `scatter_*.png` | Example outputs from a preliminary run. |

**Not included in the repo** (see `.gitignore`):
- `data/` — the 21cmFAST lightcone HDF5 files (~13 GB). Provide your own.
- `neuraloperator/` — the third-party library (see below).
- `checkpoints/` — trained model artifacts (regenerated by training).

## Environment

Python 3.12+ with PyTorch. The model depends on the
[`neuraloperator`](https://github.com/neuraloperator/neuraloperator) library
plus a few scientific packages. On a conda system:

```bash
conda install -c conda-forge pytorch numpy scipy h5py matplotlib \
    tensorly tensorly-torch opt-einsum
```

> Note: the import name `tltorch` is provided by the conda package
> **`tensorly-torch`** (there is no `tltorch` package).

### Providing `neuralop`

`fno_21cm.py` and `visualize.py` prefer a **local checkout** of the library at
`./neuraloperator/` if present, and otherwise fall back to an installed
`neuralop`. To use the vendored-source path:

```bash
git clone https://github.com/neuraloperator/neuraloperator
```

Otherwise just `pip install neuraloperator` (or the conda deps above plus the
package) and the code will use the installed copy automatically.

## Usage

```bash
# Train (expects lightcone .h5 files under ./data/)
python fno_21cm.py

# Visualize predictions from the latest checkpoint
python visualize.py
```

Key hyperparameters are constants at the top of `fno_21cm.py`
(`N_MODES`, `HIDDEN_CHANNELS`, `N_LAYERS`, `BATCH_SIZE`, `LEARNING_RATE`, ...).

## Status

Preliminary. Results so far show the model recovers large-scale ionization
morphology but struggles with sharp bubble edges and with parameter sets
outside the training distribution — partly a small-dataset / train-val-test
split artifact, partly the single-slice input not uniquely determining `x_HI`.
Active work: stratified splits, higher Fourier-mode counts, and astrophysical
parameter conditioning.
